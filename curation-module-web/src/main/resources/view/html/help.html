Curation Module is a project founded by CLARIN EU project. Its goal is to help CMD metadata authors and curators improve meta-data quality. More information about CMD framework can be found here.
Code can be found on Github.

Curation Module consists of three modules:

  • Core Module
  • Web Module
  • Link Checker Module

Core Module

It analyses individual CMD profiles, records and whole collections according to a number of quality criteria and generates reports and statistics which help discover potential problems that cause a lower metadata quality.

Web Module

It allows users to examine reports individual profiles, collections and records online.

Through Validator users can generate reports from records and profile either by their URLs or uploading them as files.

On Profiles and Collections, users can view the quality of public CMD profiles and collections respectively.

Link Checking Statistics displays the link checking results that Link Checker Module generates.

Link Checker Module

It checks constantly and repeatedly (with default delays) all the urls that collections contain. Checking in this context consists of sending HEAD and/or GET requests to URLs and saving the results in the database. Core module later uses these results to generate statistics and takes them into consideration when assessing the quality of the metadata.
